解決 Windows 上輸入指令出現「因為這個系統上已停用指令碼執行，所以無法載入...」的問題

前言

最近剛換電腦，所以很多東西和環境都要重新設定，我灌好 VS Code 後並開始寫code，因為我習慣使用 yarn 指令集，所以馬上就下了一個 yarn 來給我剛剛 clone 好的專案去安裝本身需要的套件，然後就跳出了”因為這個系統上已停用指令碼執行，所以無法載入…“這個錯誤訊息，Google 了一下解法，發現是 Set-ExecutionPolicy 的問題，這邊既然遇到了，就剛好紀錄一下。
 

問題原因

之所以會出現這個問題是因為 Windows 在執行政策上會有一定的安全性設置，其主要是避免你執行了一些不安全的指令來源，例如不小心執行了病毒指令等，因此在一般的狀況下 Windows 的執行政策 (Set-ExecutionPolicy) 預設是 Restricted，也就是「限制原則」。

而主要會發生這個問題通常都是在 Windows 10 的 PowerShell 上，官方似乎也沒有說明大概從哪一個版本開始 PowerShell 的執行原則就預設調整成 Restricted，因此若你是一名工程師的話，就會需要調整一下這個選項。
 

解法

用系統管理員身分執行，在終端機裡進行以下操作

首先可以來看看目前的執行原則，以下是指令

Get-ExecutionPolicy


 

或者可以用清單的方式去看可以更詳細了解目前的執行原則

Get-ExecutionPolicy -List


 

然後可以下 Set 去更改執行原則，把原則改成 RemoteSigned

Set-ExecutionPolicy RemoteSigned

如此一來，就可以在專案裡下各種指令去作業囉。
